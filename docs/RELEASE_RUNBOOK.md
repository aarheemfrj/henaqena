# Runbook الإطلاق والرجوع

## نشر نسخة جديدة

1. تأكد من نجاح GitHub Actions.
2. خذ نسخة PostgreSQL من `infra/backup.sh`.
3. راجع متغيرات البيئة في VPS ولا تستبدل ملف الأسرار بنسخة من Git.
4. ابنِ وشغّل الحاويات: `docker compose up -d --build`.
5. افحص `/health` و`/ready` من خارج الخادم.
6. شغّل `scripts/smoke-local.sh` بقيم `API_URL` و`WEB_URL` الإنتاجية.
7. راقب السجلات والأخطاء أول 30 دقيقة.

## الرجوع السريع

1. أوقف النشر الجديد مع إبقاء PostgreSQL وVolume الصور.
2. أعد تشغيل صورة Docker السابقة من رقم الـ commit الأخير المستقر.
3. لا تعكس تغييرات قاعدة البيانات تلقائيًا؛ استعد النسخة فقط بعد قرار واضح.
4. افحص `/ready` ثم نفّذ اختبار الدخان.

## بوابة الإطلاق

لا نطلق إلى المتاجر قبل اكتمال عناصر [RELEASE_CHECKLIST.md](./RELEASE_CHECKLIST.md)، وتجربة iOS وAndroid على أجهزة حقيقية، وتفعيل HTTPS والنسخ الاحتياطي.
